AgilFAP Material Design Bootstrap Style Guide

Esta guia de estilo foi criada para orientar Frontends / Programadores Visuais e outras áreas que no futuro tenham interesse em contribuir para a evolução do projeto.

Sobre o Projeto Repositório de Arquivos

Botões

Botões Sólidos

.btn- Os botões sólidos devem ser utilizados quando necessitam contrastar com os demais elementos de um formulário ou contexto, funcionam como objeto de ação para chamar a atenção do usuário quando necessário.

              <a class="btn btn-raised btn-primary">Primary</a>
              <a class="btn btn-raised btn-secondary">Secondary</a>
              <a class="btn btn-raised btn-success">Success</a>
              <a class="btn btn-raised btn-warning">Warning</a>
              <a class="btn btn-raised btn-danger">Danger</a>
              <a class="btn btn-raised btn-link">Link</a>
              <a class="btn btn-raised active">Active</a>

Botões Sólidos Desabilitados

.btn- Os botões desabilitados são para indicar que determinada funcionalidade não é possível ser executada e as recomendações de UX nos dizem que as mesmas só deverão ser habilitadas quando uma função puder ser executada.

<a class="btn btn-primary">Botão Primário</a>

Botões Outline

.btn- Os botões de contexto outline poderão ser utilizados em opção aos botões sólidos, desde de que a experiência do usuário não seja afetada por esta decisão de design.

<a class="btn btn-primary">Botão Primário</a>

Botões sem Borda

.btn- Os botões em borda também seguem o mesmo raciocínio dos botões outline.

<a class="btn btn-primary">Botão Primário</a>

Tamanhos

.btn- Os tamanhos que podem ser utilizados dentro do framework do projeto são os indicados abaixo.

              <button type="button" class="btn btn-raised btn-lg">Large button</button>
              <button type="button" class="btn btn-raised">Default button</button>
              <button type="button" class="btn btn-raised btn-sm">Small button</button>

Ícones

Entypo & FontAwesome

- Para o nosso projeto, 2 famílias de ícones foram selecionadas para compor o nosso repositório de representações (Entypo / Font-Awesome). Baseado em principios de web já consolidados e com famílias que permitem a ultiização dos mesmos em qualquer contexto de aplicação. Para instalar as bibliotecas basta colocar os arquivos em seus respectivos diretórios, e utilizar o arquivo de fonte de css entypo.css e o diretório /web-fonts-with-css do Fontawesome.


              <a class="btn btn-primary">Botão Primário</a>

Tipografia

Família Montserrat

- A fonte utilizada no projeto é a Montserrat, encontrada no repositório do google, para utilizar a fonte nos projetos, basta utilizar a tag: <link href="https://fonts.googleapis.com/css?family=Montserrat" rel="stylesheet" />

Formulários

Grupos de Formulários

.form-group- para agrupar coisas. Os controles de formulário individuais recebem automaticamente algum estilo global. Textual <input>, <textarea>, e <select>elementos com .form-controlestão definidos para width: 100%;por padrão. Envolva rótulos e controles no grupo .form para obter o espaçamento ideal.

<div class="form-group">
              <label>Email address</label>
              <input placeholder="hi@gmail.com" class="form-control" />
              </div>
              <div class="form-group">
              <label>Password</label>
              <input type="password" placeholder="Password" class="form-control" />
              </div>

Formulário em Linha

.form-inline- Ótimo para formulários que ocupam apenas uma linha por qualquer motivo.

@
<form class="form-inline">

                  </div>
                </div>

                  <input placeholder="Password" type="password" class="form-control" />
                </div>
                <button class="btn btn-default">Sign in</button>
              </form>

Barras de Progresso

Barra de Progresso Básica

.progress- para barras de progresso. Defina o atributo widthCSS. widthCSS attribute.

30% 
<div class="progress">
  <div style="width: 20%" class="progress-bar">
    <span class="sr-only">30%</span>
  </div>
</div>

Para de Progresso com Label

.progress- remova o .sr-onlyda classe.

<div class="progress">
  <div style="width: 22%" class="progress-bar">
    <span>22%</span>
  </div>
</div>

.progress- Barras de progresso representando porcentagens de dígito único baixo, bem como 0%, incluem min-width: 20px;para ficar mais legível.
